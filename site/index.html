Pilotez StrongSwan depuis une console web,
plus jamais depuis un fichier de conf.

StrongSwan Manager remplace l'édition manuelle de swanctl.conf et la ligne de commande par une interface claire : création guidée de tunnels IPsec, PKI intégrée, supervision temps réel, score de sécurité automatique et journal d'audit inaltérable. Tout passe par l'API VICI officielle — aucune configuration écrite à la main.

Démarrer en 3 commandes Lire la documentation 
git clone <dépôt> && cd strongswan/backend
make run
# ouvrez https://localhost:7926 — connexion : admin / admin1234
# HTTPS d'emblée (certificat auto-généré)

Le problème

StrongSwan est puissant et conforme aux standards. Mais son exploitation reste textuelle.

Configuration fragile

Éditer swanctl.conf à la main : une virgule de trop et le tunnel ne monte plus. Aucune validation avant application.

Aucune visibilité

swanctl --list-sas sur chaque passerelle, à la main. Pas de vue d'ensemble, pas d'alerte, pas d'historique.

Sécurité invisible

Un tunnel en IKEv1 / 3DES / MD5 survit des années sans que personne ne le voie. Pas de traçabilité des changements.

Ce que fait StrongSwan Manager

Chaque fonctionnalité listée ici est implémentée et utilisable dès l'installation.

Tunnels en quelques clics

Site-à-site, host-à-host, road warrior. Formulaire guidé, validation avant application, chargement à chaud via VICI. Monter, couper, renégocier depuis l'interface.

Score de sécurité automatique

Chaque tunnel est noté sur 100. IKEv1, 3DES, MD5, modp1024, PFS désactivé, absence de ML-KEM : tout est signalé, avec un bouton « Corriger ».

PKI intégrée

Autorité de certification interne, émission de certificats X.509 (ECDSA), authentification par certificat, révocation et CRL publiée sur un point de distribution.

Coffre de secrets

PSK, EAP, XAuth chiffrés au repos en AES-256-GCM. Les valeurs ne sont jamais réaffichées, ni par l'interface, ni par l'API.

Supervision temps réel

Le serveur sonde l'état des SA via VICI et pousse les changements en WebSocket. Métriques exposées au format Prometheus.

Audit inaltérable

Journal append-only garanti en base, chaîné par hachage : qui a fait quoi, quand. Toute tentative de modification est rejetée.

Versions & rollback

Chaque application de configuration crée une version horodatée et attribuée. Revenir en arrière prend un clic.

Tout StrongSwan, pas qu'un socle

Pools d'IP virtuelles, RADIUS/AAA, politiques & routage, autorités, utilisateurs VPN, règles d'alerte, paramètres du démon — tous configurables.

Rôles & permissions

Quatre rôles (administrateur, opérateur, auditeur, lecture seule). L'interface masque, l'API refuse : la lecture seule reste la lecture seule.

Comment ça marche

Le front React est embarqué dans le binaire Go et servi à la même origine que l'API. Le serveur parle à chaque passerelle via VICI, l'interface de contrôle officielle de StrongSwan — jamais en écrivant des fichiers de configuration.

Go

Backend modulaire, binaire unique, faible empreinte, compatible air-gap.

React + TypeScript

SPA embarquée, servie par le backend. Aucune dépendance runtime hors React.

PostgreSQL

Configuration, versions, PKI, audit immuable (garanti par trigger).

Docker

Une commande pour tout démarrer, y compris un lab avec de vraies passerelles.

Modèle open-core

Le cœur est libre et complet. Les modules avancés relèvent des éditions Premium et Enterprise.

Sécurité par conception

Rien n'est appliqué sans validation

Une configuration invalide ou dangereuse est refusée avant d'atteindre le démon. Aucune action destructive sans confirmation, rollback toujours possible.

Les secrets restent secrets

Chiffrement AES-256-GCM au repos. Les clés privées et les PSK ne sortent jamais de l'API, même pour un administrateur.

Traçabilité inaltérable

Journal d'audit append-only chaîné par hachage : la base elle-même refuse toute modification a posteriori.
