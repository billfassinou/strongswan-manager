StrongSwan Manager replaces hand-editing swanctl.conf and the command line with a clear interface: guided IPsec tunnel creation, built-in PKI, real-time monitoring, an automatic security score and a tamper-proof audit log. The problem

StrongSwan is powerful and standards-compliant. But operating it is still a text-file affair.

Fragile configuration

Editing swanctl.conf by hand: one stray comma and the tunnel stops coming up. Nothing is validated before it is applied.

No visibility

swanctl --list-sas on every gateway, by hand. No overview, no alerting, no history.

Invisible security

A tunnel on IKEv1 / 3DES / MD5 can survive for years without anyone noticing. And no trace of who changed what.

What StrongSwan Manager does

Every feature listed here is implemented and usable the moment you install it.

Tunnels in a few clicks

Site-to-site, host-to-host, road warrior. A guided form, validation before anything is applied, and a hot load through VICI. Bring up, tear down, rekey — all from the interface.

Automatic security score

Every tunnel is scored out of 100. IKEv1, 3DES, MD5, modp1024, PFS turned off, no ML-KEM: each one is flagged, with a Fix it button.

Built-in PKI

Internal certificate authority, X.509 (ECDSA) certificate issuance, certificate-based authentication, revocation and a CRL published on a distribution point.

Secret vault

PSK, EAP and XAuth secrets encrypted at rest with AES-256-GCM. Values are never shown again — not by the interface, not by the API.

Real-time monitoring

The server polls SA state over VICI and pushes every change over WebSocket. Metrics are exposed in Prometheus format.

Tamper-proof audit

An append-only log enforced by the database and chained by hash: who did what, and when. Any attempt to alter it is rejected.

Versions & rollback

Every configuration you apply creates a timestamped, attributed version. Going back takes one click.

All of StrongSwan, not just the basics

Virtual IP pools, RADIUS/AAA, policies & routing, authorities, VPN users, alert rules, daemon settings — all configurable.

Roles & permissions

Four roles (admin, operator, auditor, viewer). The interface hides, the API refuses: read-only really is read-only.

How it works

The React front end is embedded in the Go binary and served from the same origin as the API. The server talks to each gateway over VICI, StrongSwan's official control interface — never by writing configuration files.

Go

Modular backend, a single binary, small footprint, air-gap friendly.

React + TypeScript

An embedded SPA served by the backend. No runtime dependency beyond React.

PostgreSQL

Configuration, versions, PKI and an immutable audit log (enforced by a trigger).

systemd

A hardened service, installed by a script or a native package. No container required.

Open-core model

The core is free and complete. Advanced modules belong to the Premium and Enterprise editions.

Secure by design

Nothing is applied unvalidated

An invalid or dangerous configuration is refused before it ever reaches the daemon. No destructive action without confirmation, and rollback is always available.

Secrets stay secret

AES-256-GCM encryption at rest. Private keys and PSKs never leave the API — not even for an administrator.

Tamper-proof trail

An append-only audit log chained by hash: the database itself refuses any after-the-fact modification.
